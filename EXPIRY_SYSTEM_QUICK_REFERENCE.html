View Current Settings

SELECT p.product_id, p.product_name, ps.expiry_value, ps.expiry_unit, CONCAT(ps.expiry_value, ' ', ps.expiry_unit) AS shelf_life FROM products p LEFT JOIN production_settings ps ON p.product_id = ps.product_id ORDER BY p.product_name;

Update a Product's Shelf Life

Example: Change Fish Sauce to 30 months

UPDATE production_settings SET expiry_value = 30, expiry_unit = 'months' WHERE product_id = ( SELECT product_id FROM products WHERE product_name LIKE '%Fish Sauce%' LIMIT 1 );

Add/Insert New Product Setting

If a new product doesn't have settings:

INSERT INTO production_settings (product_id, expiry_value, expiry_unit, description) VALUES (5, 12, 'months', 'New Product Description') ON DUPLICATE KEY UPDATE expiry_value = 12, expiry_unit = 'months', description = 'New Product Description';

🔌 API Integration

API Endpoint

POST /lorinims/api/calculate_expiry_date_v2.php

Parameters

Parameter Type Required Example Notes
product_id integer ✓ Yes 4 Product ID from database
production_date string ✗ No 2026-03-01 Format: YYYY-MM-DD, defaults to today

JavaScript Example

// Call the API fetch('/lorinims/api/calculate_expiry_date_v2.php', { method: 'POST', headers: {'Content-Type': 'application/x-www-form-urlencoded'}, body: 'product_id=4&production_date=2026-03-01' }) .then(response => response.json()) .then(data => { if (data.success) { console.log('Expiry Date: ' + data.expiry_date); console.log('Shelf Life: ' + data.shelf_life_value + ' ' + data.shelf_life_unit); console.log('Source: ' + data.shelf_life_source); } else { console.error('Error: ' + data.error); } });

Response Format (Success)

{ "success": true, "expiry_date": "2028-03-01", "shelf_life_value": 24, "shelf_life_unit": "months", "shelf_life_source": "production_settings", "production_date": "2026-03-01", "error": null }

Response Format (Error)

{ "success": false, "expiry_date": null, "shelf_life_value": null, "shelf_life_unit": null, "shelf_life_source": null, "production_date": "2026-03-01", "error": "Product not found or shelf life not configured" }

⚠️ Troubleshooting

Problem: "Data truncated for column 'expiry_date'"

Cause: Invalid date values (NULL, '0000-00-00') still in database
Solution: Run the migration to clean up data
Check: Run verify_expiry_system.sql to identify invalid dates

Problem: API returns "Product not found"

Cause: Product doesn't exist OR shelf life not configured
Solution: Verify product exists, then add entry to production_settings
Check: SELECT * FROM products WHERE product_id = ?;

Problem: Expiry date calculation seems wrong

Cause: Check if using days vs months incorrectly
Solution: Verify shelf_life_unit in production_settings
Check: SELECT * FROM production_settings WHERE product_id = ?;

Problem: "Permission denied" on API

Cause: User not logged in or insufficient role
Solution: Log in, ensure user has 'admin' or 'production' role
Check: SELECT role FROM users WHERE user_id = ?;

Problem: Date format error

Cause: Production date not in YYYY-MM-DD format
Solution: Ensure form uses HTML5 date input (type="date")
Format: YYYY-MM-DD (e.g., 2026-03-01)

📚 Database Queries Reference

View All Products with Settings

SELECT p.product_id, p.product_name, COALESCE(ps.expiry_value, p.shelf_life_days, 365) AS value, COALESCE(ps.expiry_unit, 'days') AS unit, CASE WHEN ps.product_id IS NOT NULL THEN 'production_settings' WHEN p.shelf_life_days > 0 THEN 'products_legacy' ELSE 'not_configured' END AS source FROM products p LEFT JOIN production_settings ps ON p.product_id = ps.product_id ORDER BY p.product_name;

Find Products Without Shelf Life

SELECT p.product_id, p.product_name FROM products p LEFT JOIN production_settings ps ON p.product_id = ps.product_id WHERE ps.product_id IS NULL ORDER BY p.product_name;

Check Recent Batches with Expiry

SELECT pb.batch_id, pb.batch_number, p.product_name, pb.batch_date AS production_date, pb.expiry_date, DATEDIFF(pb.expiry_date, pb.batch_date) AS shelf_life_days, ps.expiry_value, ps.expiry_unit FROM production_batches pb JOIN products p ON pb.product_id = p.product_id LEFT JOIN production_settings ps ON p.product_id = ps.product_id ORDER BY pb.batch_id DESC LIMIT 20;

Calculate Days Until Expiry

SELECT pb.batch_number, p.product_name, pb.expiry_date, DATEDIFF(CURDATE(), pb.expiry_date) AS days_expired_or_remaining, CASE WHEN CURDATE() > pb.expiry_date THEN 'EXPIRED' WHEN DATEDIFF(CURDATE(), pb.expiry_date) < 30 THEN 'EXPIRING SOON' ELSE 'OK' END AS status FROM production_batches pb JOIN products p ON pb.product_id = p.product_id WHERE pb.expiry_date IS NOT NULL ORDER BY pb.expiry_date ASC;
